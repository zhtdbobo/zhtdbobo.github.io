基本使用

创建本地的git用户信息

  1. 项目级别/仓库级别
1
2

	git config user.name lyb_pro
	git config user.email jaridli_pro@163.com
  1. 操作系统用户级别
1
2

	git config --global user.name lyb
	git config --global user.email jaridli@163.com

生成ssh密钥配置到git

  1. 判断本机是否存在ssh密钥
1
2

	cd ~/.ssh
	ls
  1. 如果没有 id_rsa.pub 或 id_dsa.pub 文件，生成密钥
1

	ssh-keygen -t rsa -C "your_email@example.com"
  1. 复制.pub中的内容到github的ssh，在本地git bash进行测试
1

	ssh -T -p 443 git@github.com

本地仓库状态

  1. 添加到暂存区
1

	git add .
  1. 提交分支
1

	git commit -m "提交信息"

分支

  1. 创建分支
1

	git branch 分支名
  1. 切换分支
1

	git checkout 分支名
  1. 创建并切换分支
1

	git checkout -b 分支名

推送

  1. 连接远程仓库
    • 添加远程别名
    1

    	git remote add 别名 远程地址
    • 删除远程别名
    1

    	git remote remove 别名
    • 查看remote信息
    1

    	git remote -v
  2. 推送到远程仓库
    • 带分支名，会在远程==自动创建一个同名的分支==
    1

    	git push 别名 分支名
    • 本地有，远程没有，推送的同时在远程创建
    1

    	git push 别名 本地存在的分支名:即将创建的远程分支名
    • 不带分支名，先通过upstream和远程分支建立关联，再推送
    1
2

    	git push --set-upstream 别名 远程分支名
	git push
  3. 强制推送 
    1

    	git push -f origin master
  4. 远程仓库添加多个url同时push，pull
    • 找到.git下面的config文件
    • 添加url

Q&A

Connection refused

  1. 添加配置文件，并编辑 
    1

    vim config
  2. 添加如下内容 
    1
2
3
4
5
6

    Host github.com
User 820096913@qq.com
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443
